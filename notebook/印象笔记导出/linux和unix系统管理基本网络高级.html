系统结构

了解PC架构的一般知识，确定和配置基本的系统硬件，在尝试‘安装任何操作系统之前’你必须对这些知识有一定的了解。

  • 在PC上安装操作系统之前，必须先弄清楚系统‘安装了那些设备及相应的配置’，之后的配置则是一项后续工作
    • 必须留意的是：调制解调器，串口和并口，网卡，SCSI控制器，硬盘驱动器，USB控制器，声卡
    • 使用‘/proc文件系统’或‘一些内置命令’就可以查看当前系统的‘资源配置状况’
    • 这些设备中，尤其是老式设备，需要手动配置以避免资源冲突
  • 其余系统硬件的配置，需要在PC的固件或BIOS中进行调整，而这是一项准备工作

BIOS：负责将‘系统硬件初始化’成适合‘加载操作系统’的状态。注意：大多数BIOS厂商为各种系统架构的PC提供定制的BIOS。

  • 虽然系统各不相同，但是这个配置过程都包括：系统初始化、内存及其他设备测试、最终从几个存储设备中加载操作系统
  • BIOS提供一个低级的系统配置界面，可以让用户可以选择‘引导设备及资源分配’
    • 在引导期间，大多数PC的开机界面通常会显示进入BIOS配置程序的方法，通常是按某个特殊的按键F2
    • 一旦进入BIOS，屏幕就会出现一个菜单，让你挑选所需要的配置项目。
    • 配置菜单因厂家BIOS而定，通常会出现：磁盘，内存，板上接口（串、并），时钟等
  • 日期和时间
    • 管理主板的硬件时钟是BIOS的基本功能之一。不过，操作系统也可以设置硬件时钟；甚至在系统运行时也可以通过网络，用精确的外部时钟源（如NTPD服务器）来校正硬件时钟。
  • 硬盘和引导设备
    • BIOS另一个基本配置项是选择存储设备。较新的系统可以自动检测并恰当配置这些存储设备，但较旧的BIOS版本可能需要你自己手动配置。
    • 大多数PC至少支持三种类型的存储介质进行引导：内接式硬盘（IDE接口或SCSI接口）、CD-ROM（IDE接口或SCSI接口）以及软驱。
      • 在完成系统初始化时，BIOS会尝试在这些存储介质中寻找操作系统（或操作系统的加载程序，如LILO）
      • 许多BIOS默认从软驱或CD-ROM引导，然后是硬盘，但是这个顺序可以在BIOS进行调整
    • 除了上述默认的存储介质类型外，许多服务器主板支持从网络设备启动，如支持启动ROM的NIC。无盘工作站通常采用这种启动方式。

USB（universal serial bus）：用于连接各种外部设备的接口，包括键盘、鼠标、扫描仪、打印机等

 

改变运行级别、关机、重新启动系统

启动系统

 boot loader （如 LILO 或 GRUB）：

  • 加载Linux内核以启动系统
  • 传递‘内核参数’给Linux内核用来配置外围设备或控制其他行为

启动时的内核参数：

  • 默认情况下，系统的内核参数设置在 boot loader 的配置文件中 
  • 除此之外，也可以接受用户通过‘内核命令行’传来的参数
    • ‘内核命令行’是通过所安装的  boot loader 来实现的
    • 在系统启动时，当出现 boot loader，就可以中断默认的启动过程，为内核指定你所期望的参数。

内核启动时信息：含有‘硬件健康状态以及硬件配置’

  • 在Linux内核启动时，会将‘启动过程产生的详细的状态信息’发送到控制台（显示屏）
  • 在内核模块被加载时，也会产生状态信息

改变运行级别、关机、重新启动系统

 

运行级别：

  • 通过控制‘服务的启动顺序’来改变系统的用途
    • 所以说，运行级别的通常用途就是用于‘区分一个系统的实际用途’
  • 当Linux系统启动时，所执行的第一个进程是 init 。它将启动 所有其他的进程并且负责引导 系统进入默认的运行级别
    • 一般来说，系统的默认运行级别是2、3或5；因系统和机器而异
